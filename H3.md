# H3 Geospatial Framework — GeoETL

## Overview

H3 is Uber's hierarchical hexagonal grid system. We use it as a universal spatial index to aggregate raster (MapSPAM, SPEI) and vector (Overture/OSM) datasets onto a common grid, enabling cross-dataset joins and multi-resolution rollups.

**Resolutions used:**
- **L5** (~252 km², ~8.5 km edge) — primary working resolution, ~277K land cells globally
- **L4** (~1,770 km²) — ~50K cells, rolled up from L5
- **L3** (~12,400 km²) — ~9K cells, rolled up from L4

## Existing Modules

```
geoetl/h3/
├── grid.py           # Grid generation, loading, filtering, centroids
├── aggregation.py    # Raster→H3 zonal stats, point sampling, extensive/intensive
├── rollup.py         # H3 level rollup with derive_spec for intensive vars
└── spatial_join.py   # Polygon→H3 assignment via tiled parallel sjoin
```

### grid.py

- `generate_grid(resolution, bounds)` — tiles globe in 60° longitude strips (avoids antimeridian bug in `h3.geo_to_cells`)
- `filter_grid_to_raster(grid, raster_path)` — keeps cells whose centroid falls on a finite, non-zero raster pixel
- `h3_to_polygon(h3_index)` — converts H3 cell to shapely Polygon via `h3.cell_to_boundary()`
- `h3_to_geodataframe(h3_indices)` — batch convert to GeoDataFrame with polygon geometries
- `extract_centroids(gdf)` — adds lat/lon columns from `h3.cell_to_latlng()` (faster than geometry centroid)
- `load_grid(path)` — reads cached grid from GeoParquet or GeoJSON

### aggregation.py

- `zonal_aggregate(raster_path, h3_gdf, agg, config)` — bbox pre-filter, chunk hexagons across spawn-context workers, each worker opens raster once and masks its chunk
- `point_sample(raster_path, h3_gdf)` — nearest-neighbor centroid sampling for coarse-resolution sources (SPEI)
- `aggregate_extensive(raster_paths, h3_gdf)` — sum multiple rasters, merge on h3_index
- `derive_yield(gdf, prod_col, area_col, yield_col, scale)` — intensive variable derivation (production / area * scale)

### rollup.py

- `rollup(gdf, target_resolution, agg_spec, derive_spec)` — maps cells to parents via `h3.cell_to_parent()`, aggregates with pandas groupby, derives intensive vars post-aggregation

### spatial_join.py

- `assign_polygons(h3_gdf, polygons_gdf, columns, tile_degrees)` — tiled parallel spatial join for polygon-to-H3 assignment (admin boundaries, etc.)

## Completed: Overture Maps → H3 Pipeline (Session 5)

### Concept

Query Overture Maps GeoParquet on S3 via DuckDB, compute geometry metrics (road lengths, building areas, POI counts) per H3 cell, output to the same L5/L4/L3 grid for cross-dataset joins with MapSPAM.

### Data Source

Overture Maps releases monthly GeoParquet snapshots to S3 (no auth required):
```
s3://overturemaps-us-west-2/release/{YYYY-MM-DD.N}/theme={theme}/type={type}/*
```

Latest release: `2026-02-18.0`

Files include a `bbox` struct column (`xmin`, `ymin`, `xmax`, `ymax`) with Parquet row-group statistics, enabling DuckDB to skip irrelevant row groups via predicate pushdown.

### Overture Themes & Types

| Theme | Types | Key Fields |
|-------|-------|-----------|
| **transportation** | `segment`, `connector` | `subtype` (road/rail/water), `class` (motorway→path), `road_surface` |
| **buildings** | `building`, `building_part` | `subtype`, `class`, `height`, `num_floors` |
| **places** | `place` | `categories.main`, `confidence` |
| **base** | `land`, `land_use`, `land_cover`, `water`, `infrastructure` | Various |
| **divisions** | `division`, `division_area`, `division_boundary` | Admin boundaries |
| **addresses** | `address` | Geocoded points |

### Road Class Hierarchy

`motorway` > `trunk` > `primary` > `secondary` > `tertiary` > `unclassified` > `residential` > `living_street` > `service` > `track` > `path` > `cycleway` > `footway` > `steps`

### DuckDB Stack

Required extensions: `spatial`, `httpfs`, `h3` (community).

Key functions:
- `ST_Length_Spheroid(geom)` — linestring length in meters (geodesic, no projection needed)
- `ST_Area_Spheroid(geom)` — polygon area in sq meters (geodesic)
- `h3_latlng_to_cell(lat, lng, res)` — assign feature centroid to H3 cell in SQL
- `ST_Centroid(geom)` → `ST_Y()` / `ST_X()` for lat/lng extraction

### Query Strategy: Tile-Batched, Not Per-Cell

Group H3 cells into ~5° spatial tiles (~2,600 tiles globally). One DuckDB query per tile per theme fetches all features in the tile bbox, computes measurements, and aggregates to H3 cells — all in SQL.

Core SQL pattern:
```sql
SELECT
    h3_h3_to_string(h3_latlng_to_cell(
        ST_Y(ST_Centroid(geometry)),
        ST_X(ST_Centroid(geometry)),
        5
    )) AS h3_index,
    SUM(CASE WHEN class = 'motorway'
        THEN ST_Length_Spheroid(geometry) ELSE 0 END) AS motorway_length_m,
    -- ...per class...
    COUNT(*) FILTER (WHERE subtype = 'road') AS road_segment_count,
    SUM(ST_Length_Spheroid(geometry))
        FILTER (WHERE subtype = 'road') AS total_road_length_m
FROM read_parquet('{s3_path}/theme=transportation/type=segment/*')
WHERE bbox.xmin <= {east} AND bbox.xmax >= {west}
  AND bbox.ymin <= {north} AND bbox.ymax >= {south}
  AND ST_X(ST_Centroid(geometry)) >= {west} AND ST_X(ST_Centroid(geometry)) < {east}
  AND ST_Y(ST_Centroid(geometry)) >= {south} AND ST_Y(ST_Centroid(geometry)) < {north}
  AND subtype = 'road'
GROUP BY 1
HAVING h3_index IS NOT NULL
```

**Key detail**: Two WHERE layers — the `bbox.*` predicate enables Parquet row-group pruning (fast), and the `ST_Centroid` filter ensures each feature is counted by exactly one tile (dedup). Without the centroid filter, features near tile boundaries get double-counted.

**H3 index format**: DuckDB's `h3_latlng_to_cell()` returns integers. Wrap with `h3_h3_to_string()` to get hex strings compatible with the Python h3 library.

### Pipeline Script: `scripts/aggregate_overture.py`

| Phase | Description |
|---|---|
| 1. Tile | Generate ~5° spatial tiles (~1,944 globally, excluding Antarctica/Arctic) |
| 2. Fetch+Aggregate | Per tile × theme: DuckDB query → per-cell metrics, checkpoint per tile |
| 3. Merge | Concat tile parquets, groupby-sum dedup, outer-join themes, build geometry from H3 |
| 4. Rollup | L5→L4→L3 (extensive sums, derive avg_building_height from buildings_with_height_count) |
| 5. Validate | Conservation checks, top-cell spot checks |

CLI:
```
python scripts/aggregate_overture.py                    # Full run
python scripts/aggregate_overture.py --test             # Single tile (London area)
python scripts/aggregate_overture.py --theme transport   # One theme only
python scripts/aggregate_overture.py --skip-fetch        # Merge+rollup only
python scripts/aggregate_overture.py --release 2026-02-18.0 --tile-degrees 10
```

### Rollup Strategy

All metrics are extensive (summable) except building height:
```python
agg_spec = {col: "sum" for col in extensive_columns if col != "avg_building_height_m"}
derive_spec = {
    "avg_building_height_m": ("total_building_height_m", "buildings_with_height_count", 1.0),
}
```

Store `total_building_height_m` (SUM of heights, NULL if no data) and `buildings_with_height_count` (COUNT of buildings with non-null height) so rollup can re-derive the average correctly at each level using the correct denominator.

### Estimated Runtime

~2,600 tiles × 3 themes × ~5-15s per query = **6-12 hours** globally.
Network-bound (S3 range requests). Checkpoint/resume handles interruptions.

### Performance Notes

- `SET s3_region = 'us-west-2';` — must match bucket region
- `SET enable_object_cache = true;` — cache parquet metadata across queries
- `SET memory_limit = '16GB';` — cap DuckDB memory
- Overture parquet is spatially sorted by geohash-15 — bbox predicates get excellent row-group pruning
- One DuckDB connection per process (not thread-safe on global connection)
- Reuse connection across tiles to benefit from metadata cache

### Cross-Dataset Joins

The whole point: same H3 index across datasets.
```python
mapspam = gpd.read_parquet("outputs/mapspam/mapspam2020_h3l5.parquet")
overture = gpd.read_parquet("outputs/overture/overture_h3l5.parquet")
combined = mapspam.merge(overture, on="h3_index", suffixes=("_crop", "_infra"))
# Now: crop production + road density + building count per hex
```
